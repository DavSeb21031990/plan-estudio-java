Checked vs Unchecked exceptions strategy

Conceptos Fundamentales

Section titled “Conceptos Fundamentales”

Checked Exceptions

Section titled “Checked Exceptions”
  • Deben ser declaradas o manejadas obligatoriamente
  • DEBE declarar la excepción en el método

Unchecked Exceptions

Section titled “Unchecked Exceptions”
  • No requieren declaración obligatoria
  • NO necesita declarar RuntimeException

Comparación Detallada

Section titled “Comparación Detallada”
AspectoChecked ExceptionsUnchecked Exceptions
HerenciaExtienden ExceptionExtienden RuntimeException
CompilaciónError si no se manejanCompila sin manejo
DeclaraciónObligatoria en métodoOpcional
Manejotry/catch obligatoriotry/catch opcional
PerformanceOverhead mínimoOverhead mínimo
API DesignHace explícito el contratoAPI más limpia

Section titled “Estrategia 3: Híbrida (Recomendada)”

    • Errores de formato esperado

  • Unchecked

Criterios de Decisión

Section titled “Criterios de Decisión”

Usar Checked Exceptions cuando

Section titled “Usar Checked Exceptions cuando”
  • Errores recuperables por el caller
  • Parte fundamental del contrato de la API
  • Errores de recursos externos

Usar Unchecked Exceptions cuando

Section titled “Usar Unchecked Exceptions cuando”
  • Errores de programación
  • Errores que raramente se pueden recuperar
  • Errores del sistema

Recomendaciones por Contexto

Section titled “Recomendaciones por Contexto”

APIs Públicas

Section titled “APIs Públicas”
  • Usar checked exceptions para errores que el cliente debe manejar
  • Documentar claramente todas las excepciones
  • Mantener estabilidad en el contrato de excepciones

Código Interno

Section titled “Código Interno”
  • Preferir unchecked exceptions para mayor flexibilidad
  • Usar checked solo cuando el manejo sea realmente necesario
  • Crear jerarquías simples y coherentes

Frameworks/Librerías

Section titled “Frameworks/Librerías”
  • Seguir las convenciones del ecosistema
  • Spring: principalmente unchecked
  • Jakarta EE: mix de checked/unchecked
  • Crear wrappers apropiados para APIs externas
